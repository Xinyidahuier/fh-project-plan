第二部分 · 产品概述

项目背景、用户画像、业务流程、成功指标

2.1 项目背景与目标

Flash Home 电商项目要打造面向泰国市场的社交电商分销平台，依托 13,000+ 线下网点的客流与社群，连接卖家、网点和买家三方。网点只做推广，不囤货不碰货，按出单拿佣金，物流由 Flash Express 履约。详见《Flash Home 电商项目计划书》。

MVP 核心策略

网点推广自动化、智能化优先 — 让网点"打开就能推、一键就能发"。其余功能模块保留核心功能，以最快速度验证商业模型。

2.2 用户画像

🏪 网点（推广者）

典型画像：Flash Home 现有线下网点，每日已有稳定到店客流

核心诉求：低门槛赚取佣金，不用管库存和物流

使用场景：每日查看推荐商品 → 一键生成素材 → 分享到 LINE/Facebook → 查看佣金

🏭 卖家（中小商家）

典型画像：品牌商/制造商/贸易商，Shopee/TikTok为主要销售渠道

核心诉求：低成本开拓销售渠道，按效果付费

使用场景：上架商品 → 设置佣金 → 处理订单 → 查看收入

🛒 买家（终端消费者）

典型画像：18-60岁泰国消费者，LINE/Facebook重度用户

核心诉求：通过信任的人推荐购买，价格优惠、下单便捷

使用场景：点击朋友分享的链接 → 浏览商品 → 下单支付 → 收货

⚙️ FH 平台运营

典型画像：平台运营团队

核心诉求：高效管理商家/网点/商品，驱动网点推广效率

使用场景：配置推荐策略 → 管理推广日历 → 审核商家/网点 → 财务结算

4.1 网点端

今日推荐

今日推荐 — 网点端首页

智能推荐 + 一键分享，让网点"打开就能推"

页面构成

区域内容交互

智能推荐算法

推荐打分公式

Score = commission_rate × 0.4 + 7d_conversion × 0.3 + 7d_sales × 0.2 + freshness × 0.1

运营可在后台手动「置顶」商品，置顶商品优先展示，不参与算法排序。

因子权重数据来源说明
佣金率40%商品设置高佣金商品优先，激励网点推广
7日转化率30%订单数据点击→下单的转化表现
7日销量20%订单数据近期热销程度
新鲜度10%上架时间新品适度加权

绑定归属、佣金计算、结算、推送、推荐算法

5.1 绑定归属规则

规则说明
归因载体网点专属推广码：通用码（网点主页入口）+ 商品码（单品分享链接），系统自动追踪来源
归因窗口点击后7天 — 买家点击推广链接后7天内下单，佣金归该网点。FH品类决策周期短，7天足够覆盖
绑定触发首单自动绑定 — 买家首次通过推广码完成下单，自动与该网点建立绑定关系
绑定有效期90天滚动刷新 — 自买家最近一次交易起算90天；期间每次交易都重新刷新有效期；90天无交易则自动解绑
复购佣金全额（与首单相同）— 绑定期内买家每次复购，网点均获得与首单相同比例的推广佣金，无衰减
绑定后归属买家一旦绑定，后续订单均归属绑定网点，不受其他网点链接影响
自然流量未绑定买家直接下单→不分佣；已绑定买家直接下单→佣金仍归绑定网点

5.2 费用结构与佣金规则

FH 平台向卖家收取的费用

费用类型费率说明
成交佣金（按品类）电子配件 4% / 快消·家居·食品 5% / 美妆·保健 6%计算基数：(商品价格 − 卖家折扣) × 费率，不含运费
交易手续费3.21%支付网关费，计算基数：含平台折扣的支付金额，发货前取消免收
卖家年费฿999 / ฿2,999 / ฿9,999（入门/专业/品牌）替代逐单基础设施费；P1 前3个月首批卖家免年费（冷启动优惠）

推广佣金（卖家 → 网点）

推广佣金计算

推广佣金 = 商品售价 × 卖家自设推广佣金率

建议推广佣金率范围：10%-20%，由卖家自行设置。网点获取全额推广佣金，平台不从中抽成。

佣金率确定优先级

优先级来源说明
1（最高）单品独立佣金率商家在商品编辑页单独设置的推广佣金率
2（默认）店铺默认佣金率商家在「我的店铺」中设置的默认推广佣金率

每笔交易资金流示例（美妆品类，售价 ฿100，推广佣金 15%）

费用项费率金额去向

推送类型频率时间内容
今日推荐更新每日1次08:00推荐商品列表更新通知

推送通道（MVP）

MVP 阶段推送通过 APP 内通知（网点端推广日历） + LINE 消息推送 实现。不做短信推送。

5.5 智能推荐算法

推荐打分公式

Score = commission_rate × W1 + 7d_conversion × W2 + 7d_sales × W3 + freshness × W4

默认权重：W1=0.4, W2=0.3, W3=0.2, W4=0.1 （后台可调整）

因子计算方式归一化
commission_rate商品佣金率佣金率 / 最大佣金率 (50%)
7d_conversion近7天点击→下单转化率转化率 / 最大转化率
7d_sales近7天销量销量 / 最大销量
freshness上架时间新鲜度1 / (1 + 上架天数/30)

置顶规则：运营手动置顶的商品固定展示在列表最前面，不参与算法排序。支持设定置顶时间段（开始时间、结束时间）。

7.2 技术风险

风险影响缓解措施

高并发推送时系统压力08:00 定时推送可能导致短时高并发分批推送（5分钟窗口内随机分发）

7.3 业务风险

风险影响缓解措施
网点推广积极性不足分享率低，推广 GMV 达不到目标优化推荐算法 + 运营激励活动 + 降低分享门槛
卖家商品质量参差不齐退货率高，影响用户体验和网点信任严格商品审核 + 退货率监控 + 黑名单机制
COD 拒收率高物流成本损失监控 COD 拒收率，必要时对高拒收用户限制 COD
绑定规则引发网点争议多个网点争抢同一买家「首单绑定 + 90天滚动刷新」规则明确告知网点，保障获客者权益
